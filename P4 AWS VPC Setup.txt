STEP 5: Create Route Tables
Go to: VPC > Route Tables
5.1 Create Public Route Table

Name: my-rt-public
VPC: vpc-03bab4c2f21f169b4 (my-vpc-prac)
Tag – Key: Name | Value: my-rt-public

Click Create route table
5.2 Associate Public Subnet

Select my-rt-public > Actions > Edit subnet associations
Check: my-subnet-public-1 (subnet-09d7307a173695307)
Click Save associations

5.3 Create Private Route Table

Name: my-rt-private
VPC: vpc-03bab4c2f21f169b4 (my-vpc-prac)
Tag – Key: Name | Value: my-rt-private

Click Create route table
5.4 Associate Private Subnet

Select my-rt-private > Actions > Edit subnet associations
Check: my-subnet-private-1 (subnet-0f7c91a7779eae4ba)
Click Save associations

STEP 10: Create EC2 Key Pair
Go to: EC2 > Key Pairs > Create key pair

STEP 11: Create IAM Role for S3 Full Access
Go to: IAM > Roles > Create role
11.1 Select Trusted Entity

Trusted entity type: AWS service
Use case: S3
Use case option: S3 (Allows S3 to call AWS services on your behalf)

Click Next
11.2 Add Permissions

Search: s3
Select: AmazonS3FullAccess
Also select: AccountManagementFromVercel, AdministratorAccess-Amplify, AdministratorAccess-AWSElasticBeanstalk

Click Next
11.3 Name and Create

Role name: S3_full_access
Description: Allows S3 to call AWS services on your behalf

Click Create role
