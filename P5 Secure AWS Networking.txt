PRACTICAL 5 – Secure AWS Networking
Aim
To create a secure AWS VPC with public and private subnets, route tables, Internet Gateway, NAT Gateway, security groups, and SSH into EC2 instances.

Step 1 – Create a VPC
Go to:
AWS Console → Search VPC → Create VPC
Change region to: Mumbai (ap-south-1)
Settings:
Don't change anything
Just click Create VPC

CIDR: 10.0.0.0/16

VPC → Subnets → Create subnet

Create Private Subnet:

Step 11 – Create an IAM Role
Go to:
Search → Roles → IAM → Create role
Step 1 – Select trusted entity:
Trusted entity type: AWS service
Use case: S3
Use case option: S3 (Allows S3 to call AWS services on your behalf)
Click Next
Step 2 – Add permissions:
Search: s3
Select: AmazonS3FullAccess (AWS managed)
Also select: AccountManagementFromVercel, AdministratorAccess-Amplify,
             AdministratorAccess-AWSElasticBeanstalk
Click Next
Step 3 – Name, review and create:
Role name: S3_fullaccess_mehak
Description: Allows S3 to call AWS services on your behalf
Click Create role

Role S3_fullaccess_mehak created successfully

Step 14 – Add Inbound Rule to Public Security Group (SSH from Local Computer)
Go to:
EC2 → Security Groups → select public-sg → Inbound rules → Edit inbound rules
Add rule:
Type: SSH
Protocol: TCP
Port range: 22
Source: Custom → 172.31.19.250/32  (your local computer IP)
Click Save rules

Step 15 – Add Inbound Rule to Private Security Group (SSH from Public Subnet)
Go to:
EC2 → Security Groups → select private-sg → Inbound rules → Edit inbound rules
Add rule:
Type: SSH
Protocol: TCP
Port range: 22
Source: Custom → 172.31.19.250/32  (Public subnet CIDR or Public instance IP)
Click Save rules

Step 16 – SSH into Public EC2 Machine
From terminal / CloudShell:
bashssh -i vpc-keypair.pem ubuntu@13.200.37.100
Type yes when asked to confirm fingerprint.
Update the system:
bashsudo apt-get update
Install AWS CLI:
bashsudo snap install aws-cli --classic
List S3 buckets:
bashaws s3 ls
SSH from public EC2 into private EC2:
bashssh -i vpc-keypair.pem ubuntu@172.31.14.186
